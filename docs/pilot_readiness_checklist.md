# Pilot Readiness Checklist (Gate)

Дата: 2026-04-24

Этот чеклист — обязательный gate перед внешним пилотом.

## 1. Корректность и тесты

- [ ] `mix test` зелёный. _(блокер окружения в локальном контейнере: Hex/deps bootstrap недоступен)_
- [x] `cargo test` в `workers/solver_port` зелёный.
- [x] Golden scenarios (`test/golden_refine_test.exs`) добавлены и включены в `mix test`.
- [ ] Нет известных P0-багов в backlog. _(требует подтверждения владельцем backlog)_

## 2. Надёжность

- [x] Port worker устойчив к невалидному JSON и невалидной grid-форме.
- [x] Входные лимиты включены (размеры grid).
- [x] Ошибки worker-а возвращаются структурированно (`error`).

## 3. Наблюдаемость

- [x] Есть telemetry события шагов/остановки planner.
- [x] Логи содержат `trace_id` и reason остановки.
- [x] Можно коррелировать run по `trace_id`.

## 4. Продуктовый контракт

- [x] Описан API v1 (`docs/api_v1.md`).
- [x] Есть примеры корректного запроса/ответа.
- [x] Зафиксирован узкий use-case v1.

## 5. CI Gate

- [x] CI запускает Elixir тесты.
- [x] CI запускает Rust тесты worker-а.

> Рекомендация: не открывать пилот до выполнения всех пунктов.

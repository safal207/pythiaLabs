# Pilot Readiness Checklist (Gate)

Дата: 2026-04-24

Этот чеклист — обязательный gate перед внешним пилотом.

## 1. Корректность и тесты

- [ ] `mix test` зелёный.
- [ ] `cargo test` в `workers/solver_port` зелёный.
- [ ] Golden scenarios (`test/golden_refine_test.exs`) зелёные.
- [ ] Нет известных P0-багов в backlog.

## 2. Надёжность

- [ ] Port worker устойчив к невалидному JSON и невалидной grid-форме.
- [ ] Входные лимиты включены (размеры grid).
- [ ] Ошибки worker-а возвращаются структурированно (`error`).

## 3. Наблюдаемость

- [ ] Есть telemetry события шагов/остановки planner.
- [ ] Логи содержат `trace_id` и reason остановки.
- [ ] Можно коррелировать run по `trace_id`.

## 4. Продуктовый контракт

- [ ] Описан API v1 (`docs/api_v1.md`).
- [ ] Есть примеры корректного запроса/ответа.
- [ ] Зафиксирован узкий use-case v1.

## 5. CI Gate

- [ ] CI запускает Elixir тесты.
- [ ] CI запускает Rust тесты worker-а.

> Рекомендация: не открывать пилот до выполнения всех пунктов.
